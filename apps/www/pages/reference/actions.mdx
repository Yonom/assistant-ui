Actions are helpers based on the Context API that allow you to perform common actions.

## Thread Actions

### `useApplyThreadSuggestion`

Applies a suggestion to the current thread composer. This is the same behavior as `ThreadPrimitive.Suggestion`.

Not available when the thread is running.

```tsx
import { useApplyThreadSuggestion } from "@assistant-ui/react";

const Suggestion = () => {
  const applySuggestion = useApplyThreadSuggestion({
    prompt: "Hello",
    method: "replace",
    autoSend: true,
  });

  // apply suggestion action is not available
  if (!applySuggestion) return null;

  return <button onClick={applySuggestion}>Apply Suggestion</button>;
}
```

### `useScrollToThreadBottom`

Scrolls to the bottom of the thread. This is the same behavior as `ThreadPrimitive.ScrollToBottom`. 

Not available when the thread viewport is already scrolled to the bottom.

```tsx
import { useScrollToThreadBottom } from "@assistant-ui/react";

const ScrollToBottom = () => {
  const scrollToBottom = useScrollToThreadBottom();

  // scroll to bottom action is not available
  if (!scrollToBottom) return null;

  return <button onClick={scrollToBottom}>Scroll to Bottom</button>;
};
```


### `useCancelThreadRun`

Cancels the current thread run. This is the same behavior as `ComposerPrimitive.Cancel` inside a `Thread`.

Not available when the thread is not running.

```tsx
import { useCancelRun } from "@assistant-ui/react";

const Cancel = () => {
  const cancel = useCancelRun();

  // cancel action is not available
  if (!cancel) return null;

  return (
    <button onClick={cancel}>Cancel</button>
  );
};
```


### `useThreadIf`

Runs a query on the current thread state and returns a boolean. This is the same behavior as `ThreadPrimitive.If`.

```tsx
import { useThreadIf } from "@assistant-ui/react";

const isRunning = useThreadIf({ running: true });
```

## Message Actions

### `useCopyMessage`

Copies the current message to the clipboard. This is the same behavior as `ActionBarPrimitive.Copy`.

Not available when the message has no text content or if the message is being edited.

```tsx
import { useCopyMessage } from "@assistant-ui/react";

const Copy = () => {
  const copy = useCopyMessage({ copiedDuration: 3000 });

  // copy action is not available
  if (!copy) return null;

  return (
    <button onClick={copy}>Copy</button>
  );
};
```

### `useReloadMessage`

Reloads the current assistant message. This is the same behavior as `ActionBarPrimitive.Reload`.

Not available when the thread is running or if the message is not an assistant message.

```tsx
import { useReloadMessage } from "@assistant-ui/react";

const Reload = () => {
  const reload = useReloadMessage();

  // reload action is not available
  if (!reload) return null;

  return (
    <button onClick={reload}>Reload</button>
  );
};
```

### `useBeginMessageEdit`

Transitions the current user message to the edit mode. This is the same behavior as `ActionBarPrimitive.Edit`.

Not available when the message is not a user message or if the message is already in edit mode.

```tsx
import { useBeginMessageEdit } from "@assistant-ui/react";

const Edit = () => {
  const edit = useBeginMessageEdit();

  // edit action is not available
  if (!edit) return null;

  return (
    <button onClick={edit}>Edit</button>
  );
};
```

### `useCancelMessageEdit`

Cancels the current user message edit. This is the same behavior as `ComposerPrimitive.Cancel` inside a `Message`.

Not available if the current message is not in edit mode.

```tsx
import { useCancelMessageEdit } from "@assistant-ui/react";

const Cancel = () => {
  const cancel = useCancelMessageEdit();

  // cancel action is not available
  if (!cancel) return null;

  return (
    <button onClick={cancel}>Cancel</button>
  );
};
```

## Composer Actions

### `useSendComposer`x

Sends the current message. This is the same behavior as `ComposerPrimitive.Send`.

Not available when the composer content is empty.  

If used inside a `Message`: equivalent to `useSendMessageEdit`, not available when the message is not in edit mode.
If used outside of a `Message`: equivalent to `useSendMessage`, not available when the thread is not running.

```tsx
import { useSendComposer } from "@assistant-ui/react";

const Send = () => {
  const send = useSendComposer();

  // send action is not available
  if (!send) return null;

  return (
    <button onClick={send}>Send</button>
  );
};
```

### `useCancelComposer`

Cancels the current composer. This is the same behavior as `ComposerPrimitive.Cancel`.

If used inside a `Message`: equivalent to `useCancelMessageEdit`, not available when the message is not in edit mode.
If used outside of a `Message`: equivalent to `useCancelThreadRun`, not available when the thread is not running.

```tsx
import { useCancelComposer } from "@assistant-ui/react";

const Cancel = () => {
  const cancel = useCancelComposer();

  // cancel action is not available
  if (!cancel) return null;

  return (
    <button onClick={cancel}>Cancel</button>
  );
};
```

### `useComposerIf`

Runs a query on the current composer state and returns a boolean. This is the same behavior as `ComposerPrimitive.If`.

```tsx
import { useComposerIf } from "@assistant-ui/react";

const isEditing = useComposerIf({ editing: true });
```
